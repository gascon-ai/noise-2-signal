What you're working with

Read this first. Just enough to know what you are working with. You will mostly live in Claude Code, at minimum Cowork. The rest is context.

These are the ways to use Claude, from just talking to doing the work. Each step adds a building block.

  • Claude Chat = a place to talk. Ask, get an answer. It can search the web, read a file you drop in, and remember across chats. Like a brilliant generalist you can text anytime. It is not sitting at your computer doing your filing.
  • Claude Projects = a place + rules + reference. A workspace for one topic with standing instructions and a knowledge base you add once, so every chat already knows them. Like a labeled binder for one client or job. Still chat, just with your context built in.
  • Cowork = a place + rules + tools. Claude steps out of the chat and works across your apps, taking steps while you watch. Like a teammate sharing your screen.
  • Claude Code = all of it, on your real files. It reads and writes your actual files, runs commands, connects your tools, and keeps what it learns. Like an employee who sits at your computer and does the work.
  • Other features: artifacts, web search and research, voice, file and image analysis, connectors to your apps. Useful extras.

Which to use

The question everyone asks. The rule:

  • Answer it in Claude Chat. A quick question, a brainstorm, a one-off draft.
  • Remember it in a Project. Ongoing work on one topic that needs the same context every time.
  • Do it with you in Cowork. A multi-step task in your apps you want done while you watch.
  • Build and run it in Claude Code. Anything that touches your real files or has to last.

Why Anthropic, not OpenAI or the rest

They can all code. The difference is trust and taste: Claude follows your rules tightly, admits when it is unsure instead of bluffing, and writes like a human, not slop. Anthropic's whole business is safe, reliable AI, not maximizing your screen time. When you hand a system the keys to your business, you want the most trustworthy one that is also the best at agentic work. That is Claude.

Key takeaway: a Project is a place plus rules. Claude Code is that same thing on your real files, with hands.

Install and run Claude Code

Outcome: Claude Code running, signed in, and you know which way you are using it.

Three ways to run it, easiest to most powerful:

  • Web (claude.ai/code), beginner. Pro: nothing to install, start in seconds from any device. Con: it is a cloud sandbox that resets. Fix: connect it to your GitHub so it works with your real files in the cloud.
  • Desktop app, intermediate. Pro: a real app on your Mac that works with your local files, no terminal needed. Con: it can be buggy and glitchy, and you see multiple sessions one at a time.
  • Terminal, advanced. Pro: the full power, and you can run several sessions at once and see them all on one screen, plus it pairs with your editor like Cursor. Con: it takes a bit of getting used to. Install Node with Homebrew, then npm install -g @anthropic-ai/claude-code.

Do this

  1. Pick your way in. Web to try it, terminal to run your business on it.
  2. Sign in with your Claude account. You do not need an API key. (That is only for usage-based billing or headless automation.)
  3. Verify it works: run claude --version.
Download · the install + first-run checklist
Done when: claude --version returns a version and you are signed in.

The setup I recommend: start with the desktop app so it reaches your real files, set up GitHub for backup and sync, and connect the web to your GitHub so you can work from the cloud too. Graduate to the terminal next to Cursor when you want to run several sessions at once.

Permissions and slash commands

Outcome: you control what the AI does on its own, and you keep it sharp by managing its memory.

Permissions: your seatbelt

How much Claude can do without asking. Four settings:

  • Ask first: it checks before every edit and command. Safest, for unfamiliar work.
  • Accept edits: it edits files on its own, still asks before running commands. Start here.
  • Plan mode: read-only, it proposes a plan before doing anything. The creator of Claude Code starts every project here. More in 1.6.
  • Bypass: it asks for nothing and runs everything. Powerful and risky. Only when you trust the task, never loose on your main machine.

Hack: tell it your intention. "My stated intention is X, because the thing I want true is Y." It aims better when it knows why.

Slash commands: your buttons

First the why. The AI is a 200-IQ genius with amnesia. The context window is how much of the conversation it can hold, and the longer a chat runs the worse it gets, it starts to be confidently wrong. The rule: one task, one window.

Type a slash for your buttons. The daily five:

  • /context how full the window is.
  • /clear wipe it, fresh start. New task, clear.
  • /compact summarize and keep the essence without the rot. Same task gone long, compact.
  • /model switch the brain. Smart for hard work, fast for simple.
  • /cost what this session is costing.

The big one: you can make your own slash commands. Every workflow you repeat becomes one button. That is skills, built in Power features.

Do this

  1. Set permissions to Accept edits.
  2. Run a task, then use /clear before the next unrelated one.
  3. When a session runs long, use /compact instead of pushing through.
Download · the permissions + slash command cheat sheet
Done when: you can set your permission level and keep one task to one window.

Key takeaway: permissions are the seatbelt, slash commands are the buttons, and one task one window keeps Claude sharp.

Git and GitHub

Outcome: your workspace is backed up and versioned, so you can never truly lose work.

Git takes a snapshot of your folder every time you save (a commit), so you can always go back. GitHub is the same thing in the cloud, a private copy that is the real one, the master. Your computer is a working copy. You do not learn git commands, Claude Code does it. You just keep the habit: save a version, push it up.

Do this

  1. Turn your home folder into a git repository (Claude Code walks you through it).
  2. Create a private GitHub repo and connect it.
  3. Save your first version and push it up.
Download · git + GitHub setup walkthrough
Done when: your folder shows up in your private GitHub repo.

Key takeaway: GitHub is the master, your computer is a copy, your chat window is nothing. If it matters, it is committed.

Your rule book and plan mode

Outcome: one file the AI reads every time, plus a way to make it plan before it acts.

Remember Claude Projects, standing instructions plus a knowledge base? CLAUDE.md is that same idea, now in your real files instead of inside chat. It is your house rules: a plain file at the top of your workspace the AI reads every session. How you write, what to never do, where things live. Write the rule once and every session follows it. That is the difference between a tool and an employee.

Plan mode is the other half. For anything big, the AI lays out the plan, you approve, then it builds. No surprises.

Do this

  1. Drop the starter CLAUDE.md at the top of your workspace.
  2. Add your non-negotiables (for example: no em dashes, always ask before sending anything).
  3. Turn on plan mode and watch it propose before it touches anything.

Capture everything

Outcome: one inbox where every idea, note, and loose end lands, so none of it lives in your head.

This is the Getting Things Done principle, and it is the cure for a million tabs open in your head: your mind is for having ideas, not holding them. You set up one capture spot. An idea on your phone, a thought mid-call, a to-do, it all goes to the same place, fast, then gets sorted later.

Do this

  1. Set up one capture inbox in your workspace (a single note or a quick-capture command).
  2. Practice the habit: capture first, sort later. Do not organize in the moment.
  3. Empty the inbox on a schedule (the weekly review comes later).

The data architecture

Outcome: you know where every kind of thing lives, what is the master, and how to never lose data.

The lesson most people skip and later regret. Every piece of data has one of three roles. The master is the real, true copy. A copy is a working version you edit freely. A cache is throwaway, something you can always regenerate. The whole trick is knowing which is which, so you never destroy the only real copy.

Your map: GitHub is the master, your computer is a working copy, your databases and notes are a cache or system of record, the cloud is cold storage. The discipline: never move or delete the only copy of anything. Copy it, confirm it landed, then remove the original.

One trap because it is the most common: never keep your working folder inside iCloud Drive. iCloud swaps files for placeholders to save space and silently breaks git, your tools, and the AI reading your files. Home folder only.

Do this

  1. Fill in the data-architecture map for your setup.
  2. Name your master for each kind of thing (files, money data, content data).
  3. Adopt the discipline: copy, verify, then delete. Never delete the only copy.

Done when: you can point to the master for every kind of data, and nothing critical lives only in iCloud.
