Permissions and slash commands

Outcome: you control what the AI can do on its own, and you know how to trigger saved actions.

Two things make Claude Code safe and fast. Permissions decide what it can do without asking you first, that is your seatbelt. Start cautious: it asks before it changes or sends anything. As you build trust, you loosen it. Never the other way around.

Slash commands are your buttons. Type a forward slash and you see your saved moves. Right now there are only a few. Later, every skill you install shows up here, so a whole workflow runs from one command.

Do this

  1. Set your permission mode to the cautious default (it asks before acting).
  2. Type / and look at the commands available today.
  3. Run one read-only command to feel it work.
Download · starter permission settings
Done when: you have run one command and nothing changed without your say-so.

Key takeaway: permissions are the seatbelt, slash commands are the buttons. Loosen the belt slowly.

Your file system

Outcome: one folder that holds your whole business, laid out so nothing gets lost.

Your second brain lives in files, in one folder on your computer. Not the Desktop, not iCloud Drive (iCloud quietly swaps your files for placeholders and breaks things, more on that in the data architecture lesson). Your home folder.

Inside, the layout mirrors this system: a place for your brain (your identity and memory), and a place for each department (content, outreach, sales, clients, finance, admin). One naming rule, lowercase with hyphens, no spaces. Boring and consistent beats clever every time.

Do this

  1. Create one home folder for your business in your home directory.
  2. Drop in the folder-structure template below and let Claude Code build out the rooms.
  3. Apply the naming rule to everything from here on.

Git and GitHub

Outcome: your workspace is backed up and versioned, so you can never truly lose work.

Git takes a snapshot of your folder every time you save (a commit), so you can always go back. GitHub is the same thing in the cloud, a private copy that is the real one, the master. Your computer is just a working copy.

You do not need to learn git commands. Claude Code does it for you. You just need to understand the habit: save a version, push it up. Do it once by hand so it sticks, then it happens in the background.

Do this

  1. Turn your home folder into a git repository (Claude Code walks you through it).
  2. Create a private GitHub repo and connect it.
  3. Save your first version and push it up.
Download · git + GitHub setup walkthrough
Done when: your folder shows up in your private GitHub repo.

Key takeaway: GitHub is the master, your computer is a copy, your chat window is nothing. If it matters, it is committed.

Your rule book and plan mode

Outcome: one file the AI reads every time, plus a way to make it plan before it acts.

CLAUDE.md is your house rules. It is a plain file at the top of your workspace that the AI reads every single session: how you write, what to never do, where things live. Write the rule once and every session follows it. That is the difference between a tool and an employee.

Plan mode is the second half. For anything big, the AI lays out the plan first, you approve it, then it builds. No surprises, no runaway changes.

Do this

  1. Drop the starter CLAUDE.md template at the top of your workspace.
  2. Add your non-negotiables (for example: no em dashes, always ask before sending anything).
  3. Turn on plan mode and watch it propose a plan before it touches anything.

Lesson 7video

The data architecture

Outcome: you know where every kind of thing lives, what is the master, and how to never lose data.

This is the lesson most people skip and later regret. Every piece of data has one of three roles. The master is the real, true copy. A copy is a working version you can edit freely. A cache is throwaway, something you can always regenerate. The whole trick is knowing which is which, so you never destroy the only real copy of something.

Here is the map you are setting up. GitHub is your master. Your computer is a working copy. Your databases and notes are a cache or a system of record. The cloud (Google Drive, Dropbox) is cold storage. And the discipline that ties it together: never move or delete the only copy of anything. Copy it, confirm it landed, then remove the original.

One specific trap, because it is the most common: never keep your working folder inside iCloud Drive. iCloud swaps your files for tiny placeholders to save space, and that silently breaks git, your tools, and the AI's ability to read your files. Home folder only.

Do this

  1. Fill in the data-architecture map template below for your setup.
  2. Name your master for each kind of thing (files, money data, content data).
  3. Adopt the one discipline: copy, verify, then delete. Never delete the only copy.
Download · data-architecture map template
Done when: you can point to the master for every kind of data you own, and nothing critical lives only in iCloud.

Key takeaway: most people lose work because they never decided what the master is. You just did.
