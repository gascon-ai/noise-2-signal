Skills

Outcome: a packaged, repeatable job the AI runs the same way every time, and the ability to build your own.

A skill is a job written down once so it never has to be re-explained. Instead of typing the same long prompt every time you want a cold email or a renewal pack, you teach the system the steps, the rules, and the files it needs. Then you call it by name and it runs.

This is the genius with amnesia getting muscle memory. The brain stays the same, but now it knows how to do specific things on command, the way you would do them.

You install skills people have already built, and you build your own. The build is easy, because there is a skill that writes skills.

Do this

  1. Install one skill and run it, so you feel the call-by-name move.
  2. Take a job you do the same way every week and write down the steps.
  3. Hand those steps to the skill-creator and let it package the skill for you.
Download · the skill-creator
Done when: you have one skill you built running a real, repeatable job in your business.

Key takeaway: a skill turns a long prompt into a one-word command. Receipt: 35 skills already run this business, from cold email to client renewals.

MCP, connect your tools

Outcome: the genius gets hands in your real accounts, not just a chat window.

Skills tell the AI how to do a job. MCP gives it the place to do it. MCP, the Model Context Protocol, is the standard plug that lets Claude reach your actual tools and act in them: Notion, Slack, Gmail, Calendar, Stripe, Apify, your CRM.

Before this, the AI could write you a perfect Slack message. After this, it posts it. It reads the Notion board, pulls the Stripe invoice, checks the Calendar, scrapes with Apify, all in your real accounts, with your data. The amnesiac genius finally has hands on the controls.

You connect one tool at a time, and you only connect what you use. Each one is a new thing the system can touch.

Do this

  1. List the tools you live in. Notion, Slack, Gmail, Calendar, Stripe, your CRM.
  2. Connect one, the one you touch most, and confirm it reads your real data.
  3. Add the rest over time, one per session, never all at once.
Download · the MCP connect checklist
Done when: Claude reads and acts in at least one of your real accounts, not a copy you pasted in.

Key takeaway: skills are how it works, MCP is where it works. Together the brain gets hands.

Subagents

Outcome: one task fans out to parallel workers, so no single window gets clogged.

Remember the one rule: one task, one window. A subagent is how you keep that rule while doing more at once. Instead of cramming five jobs into one chat until it loses the thread, you fan the work out. Each subagent gets its own clean context, does its piece, and returns only the answer.

Say you want five competitors researched. One window doing all five gets muddy and slow. Five subagents, one per competitor, each comes back with just its findings. Your main window stays clean and collects the results.

This is the difference between a smart assistant and a small team. You stay the one giving the orders.

Do this

  1. Find a task that splits cleanly into parts, like research across several targets.
  2. Use the fan-out prompt to send one subagent per part.
  3. Read the answers they return, not the work they did to get there.
Download · the subagent fan-out prompt
Done when: a job that would have clogged one window comes back as clean answers from several workers.

Key takeaway: subagents let one task become many, each in its own clean context, so you scale without losing the thread.

Hooks

Outcome: code that fires automatically around a session, so the system does work you never have to remember.

A hook is a small piece of code that runs on its own at a set moment: when a session starts, when you run a command, when a session ends. You set it once and it fires every time, no prompting, no clicking.

Here is the real receipt. A Stop hook runs the moment any session ends. After every session, it syncs the whole Claude setup, the skills and the memory, to a private GitHub repo. The work backs itself up with zero effort. You never think about it, and you never lose it.

That is the shape of a good hook. A chore you would forget, handed to a trigger that never forgets.

Do this

  1. Pick a chore that should happen every time, like backing up your setup.
  2. Install the auto-backup hook on session end and point it at a private repo.
  3. End a session and confirm the backup landed without you touching anything.
Download · the auto-backup hook
Done when: your skills and memory back themselves up to GitHub every time a session ends, untouched.

Key takeaway: hooks turn a chore into a reflex. Set it once, it fires forever.

Plan mode and the command library

Outcome: the system shows its plan before it touches anything, and your best prompts live as one-word commands.

Two habits that keep a powerful system safe and fast.

Plan mode makes it think before it acts. It reads, it lays out exactly what it intends to do, and it waits for your yes before changing a single file. You catch the wrong turn while it is still just words. On anything that touches real work, plan first, then run.

The command library is where your best prompts retire. Any prompt you find yourself typing twice becomes a slash command. The handoff, the research fan-out, the weekly report, saved by name and called in one word. It pairs with the one rule: one task, one window, one command to start it clean.

Do this

  1. Turn on plan mode and make it show the plan before any real change.
  2. Find a prompt you reuse and save it as a slash command.
  3. Load the starter library so you begin with a shelf of commands, not an empty one.
Download · the starter command library
Done when: it plans before it touches your work, and your top prompts run as one-word commands.

Key takeaway: plan mode is the brake, the command library is the gas. One task, one window, one command.

Agents as AI employees

Outcome: the pieces from this module combine into a worker you can hand a whole job to.

Here is where it all comes together. A skill is how to do a job. Memory is who you are and what the job is about. MCP is the hands in your real tools. Stack those three and you get an agent: a worker you can hand a job to and trust to come back with it done.

An AI employee is not a chatbot. It has a role, it knows your business from memory, it runs its job as a skill, and it acts in your accounts through MCP. You delegate the outcome, not the steps.

This is the bridge out of the stack. You have built the house, furnished it, and given it hands. Next you stand up the business, one function at a time, and these workers become your departments: content, outreach, sales, client success, finance.

Do this

  1. Pick one job in your business you would hand to a hire.
  2. Give a worker its three pieces: a skill, the memory it needs, the tools it acts in.
  3. Hand it the outcome and check the result, not the steps.
Download · the AI employee blueprint
Done when: one agent owns a real job end to end, and you judge it by the result.

Key takeaway: skill plus memory plus tools equals an employee. This is the bridge: next, these become your departments.
